# Eric Black

Cloud and disaster recovery architect building operator-focused tooling for migration planning, recovery readiness, platform automation, and lifecycle operations.

I focus on the layer between architecture and execution: discovery before migration, evidence before recovery claims, and automation that produces auditable outputs for the teams who have to run it.

## What I build

- Mixed-estate discovery and migration planning across virtualization and backup platforms
- Disaster recovery assessment, scoring, and reporting for Kubernetes and data protection workflows
- PowerShell automation for Hyper-V, vSphere, and Windows infrastructure
- Operator-facing tools that turn platform state into usable reports, workflows, and safer change decisions

## Featured projects

- [Viaduct](https://github.com/eblackrps/Viaduct) - Open-source control plane for mixed virtualization discovery, dependency-aware planning, and supervised pilot migration workflows. `Go · React · PostgreSQL`
- [HyperVClusterPlatform](https://github.com/eblackrps/HyperVClusterPlatform) - Production-hardened Hyper-V cluster automation with audit, enforce, remediate, health scoring, and fleet orchestration workflows. `PowerShell · Pester · Windows Server`
- [k8s-recovery-visualizer](https://github.com/eblackrps/k8s-recovery-visualizer) - Kubernetes disaster recovery assessment tool that scores cluster readiness and generates offline operator reports. `Go · HTML/Markdown reporting · JSON policy outputs`
- [veeam_designer](https://github.com/eblackrps/veeam_designer) - Veeam environment sizing and architecture designer with a Python engine and FastAPI web UI. `Python · FastAPI · YAML`
- [mcp-home](https://github.com/eblackrps/mcp-home) - Read-only MCP server for Windows homelab operations across Docker, Plex, files, repos, and health dashboards. `TypeScript · MCP`

## Selected utilities

- [recovery-risk-translator](https://github.com/eblackrps/recovery-risk-translator) - Translates recovery objectives and incident inputs into executive-readable DR risk statements and recommendations.
- [ItsDeadJim](https://github.com/eblackrps/ItsDeadJim) - Read-only chaos simulation tool for modeling backup failure blast radius before it shows up in production.

## How I build

- Operator first: clear inputs, safe defaults, and outputs people can inspect, export, and use
- Conservative by design: recovery claims should be backed by evidence, not assumed from tool presence alone
- Automation with context: discovery, scoring, and reporting come before enforcement
- Practical over theatrical: tools should help teams make decisions and execute safer changes
